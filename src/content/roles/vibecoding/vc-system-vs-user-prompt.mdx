## Зачем

System prompt — постоянное (роль, стиль, запреты, кодовые конвенции). User prompt — текущая задача. У модели есть иерархия инструкций: system весит больше user. Если запихнуть конвенции в user — они растворятся в задаче и могут быть «забыты». CLAUDE.md в современных инструментах работает как фактический system prompt для проекта.

## Ключевые различения

- System = «как ты работаешь всегда» ↔ user = «что делаешь сейчас»
- Persistent instructions (вес высокий) ↔ ad-hoc (вес низкий)
- Output format (JSON, markdown) лучше задать в system
- Tool restrictions = system-level

## Инструменты

- CLAUDE.md как фактический project-level system prompt
- Tool permissions в settings.json — system constraint
- settings.local.json — для личных override без коммита

## Практика

- Пиши и поддерживай CLAUDE.md (project conventions)
- Новое правило → внеси в CLAUDE.md, не повторяй в каждом chat
- Output формат, стиль коммитов, запреты — в CLAUDE.md один раз

## Маркеры освоения

- Заметил, что повторяешь одно и то же в трёх разных сессиях → добавил в CLAUDE.md
- Знаешь, какие конвенции в твоём CLAUDE.md и почему

## Антипаттерны

- Повторять «не делай X» в каждом сообщении
- Полагаться только на пользовательский prompt
- Не иметь CLAUDE.md в проекте, который AI редактирует

## Где в стеке

**Уровень 1: Вход в контекст.** AI не знает, чего вы хотите, и не видит репо целиком. Вход = задача + контекст; качество выхода ограничено качеством входа.
