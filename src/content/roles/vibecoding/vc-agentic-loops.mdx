## Зачем

Современный AI-ассистент не выдаёт «один большой ответ». Он работает циклом: получает задачу → строит план → вызывает инструменты (read, edit, run tests) → читает результат → корректирует → следующий шаг. Это agentic loop. Чем больше шагов, тем больше нужен контроль: явный план до действий, чтение результата каждого tool-вызова, остановка при отклонении от плана.

## Ключевые различения

- Chat-completion ≠ agentic (chat = один turn, agentic = N turns с tools)
- Tool use = действие в мире (запись файла, запуск команды)
- План до действий = проверяемость (что сделает прежде, чем сделает)
- Self-correcting loop ↔ doom loop (зацикливание)

## Инструменты

- Plan mode (Claude Code) — план без действий до утверждения
- Cursor agent mode, Aider — варианты того же цикла
- Logs действий: что было прочитано, что записано, что запущено

## Практика

- Для нетривиальной задачи запрашивай план до действий
- Читай каждый tool-вызов в логе, особенно edit и exec
- Останавливай цикл, если план уходит не туда

## Маркеры освоения

- Используешь plan-mode для задач сложнее «изменить строку»
- Знаешь, как остановить агента и переформулировать без перезапуска сессии
- В долгих сессиях не теряешь нить — следишь за state'ом

## Антипаттерны

- «Выполни» без чтения плана
- Ждать «пока сам закончит» 30 минут на одной задаче
- Не читать exec output — «всё же зелёное»
