## Зачем

Tool — это «руки» AI: read, edit, exec, web fetch, package install, deploy. Хороший дизайн = разрешено всё что обратимо (правка файлов с git diff, локальный exec в worktree), требует approval всё что необратимо (push в main, deploy, delete branch, ALTER на prod БД). Принцип: blast radius должен быть локальным; глобальное действие — через человека.

## Ключевые различения

- Обратимое (правка файла, локальный exec) ↔ необратимое (push, deploy, DROP)
- Read vs write vs exec — разные уровни доверия
- Локальный ↔ внешний (вызов чужого API, отправка email)
- Tracked в git ↔ untracked (любые изменения, которые git diff не увидит — подозрительны)

## Инструменты

- settings.json: permissions allow/deny списки
- .cursorignore / .gitignore — секреты вне зоны AI
- Approval prompts для опасных tools (Vercel CLI's auto-mode classifier — пример)

## Практика

- Перед запуском новой задачи: «что в худшем случае?» — если необратимо, требуй approval
- Заведи allowlist для безопасных команд (pnpm test, git status) — без подтверждения
- Чувствительные операции — за человеком, без исключений
