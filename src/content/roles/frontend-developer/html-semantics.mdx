## Что вы реально получаете от «семантики»

Две страницы могут выглядеть пиксель в пиксель одинаково и при этом
ощущаться совершенно по-разному во вспомогательных технологиях. Первая
использует `<div>` для всего; вторая — правильные landmark'и. У скринридера
нет способа пропустить навигацию на первой странице — каждый абзац читается
с самого начала. На второй пользователь одной клавишей перепрыгивает к
`<main>`.

В этом и суть: **каждый семантический тег — это подсказка браузеру,
поисковикам и вспомогательным технологиям**. Голый `<div>` говорит: «Я не
знаю, что это такое».

## Набор, который действительно важен

| Тег | Для чего |
|---|---|
| `<header>` | Преамбула страницы или секции: логотип, заголовок, вводная навигация. |
| `<nav>` | Основные навигационные ссылки. Одна главная nav на страницу; второстепенные тоже допустимы. |
| `<main>` | Уникальный контент страницы (один на документ). |
| `<article>` | Самостоятельное, независимо распространяемое содержание. |
| `<section>` | Тематическая группировка внутри документа. Нужен заголовок. |
| `<aside>` | Косвенно связанный контент — сайдбар, врезка, связанные ссылки. |
| `<footer>` | Заключение страницы или секции: копирайт, контакты, ссылки. |

## Типичный рефакторинг

Было:
```html
<div class="header">
  <h1>My Blog</h1>
  <div class="nav"><a href="/">Home</a></div>
</div>
<div class="post">…</div>
<div class="footer">©</div>
```

Стало:
```html
<header>
  <h1>My Blog</h1>
  <nav><a href="/">Home</a></nav>
</header>
<article>…</article>
<footer>©</footer>
```

Те же пиксели. Заметно лучшая доступность, а имена CSS-классов начинают
описывать **что**, а не **где**.

## Если сомневаетесь

- Повторяющаяся «обёртка» страницы? `<header>` / `<footer>` / `<nav>`.
- Самостоятельный блок, который можно отдать по RSS? `<article>`.
- Раздел длинного документа с заголовком? `<section>`.
- Уникальный контент страницы? `<main>` (ровно один).
- Косвенно связанное, без чего страница не сломается? `<aside>`.
