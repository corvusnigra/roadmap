## What happens when you press Enter

When you type `https://example.com/articles` into the address bar and hit Enter, your
browser doesn't magically reach the server. It runs a multi-step pipeline that's
useful to picture as one mental model:

1. **DNS** — the browser asks "what's the IP of `example.com`?" Operating system caches the answer
   for a while, otherwise queries a DNS resolver (often your ISP's, or `1.1.1.1`/`8.8.8.8`).
2. **TCP** — the browser opens a reliable connection to that IP on port 443 (HTTPS) or 80 (HTTP).
3. **TLS** — for HTTPS the browser and server perform a TLS handshake, agreeing on encryption
   keys and verifying the server's certificate.
4. **HTTP** — the browser sends an HTTP request like:
   ```
   GET /articles HTTP/1.1
   Host: example.com
   Accept: text/html
   ```
5. **Response** — the server sends back a status line, headers, and a body:
   ```
   HTTP/1.1 200 OK
   Content-Type: text/html; charset=utf-8
   Content-Length: 12048

   <!doctype html>...
   ```
6. **Render** — the browser parses the HTML, requests every linked asset (CSS, JS, images,
   fonts), and progressively paints the page.

## Why the layering matters

Each layer has a single job:

- **DNS** is a phonebook — only maps names to addresses.
- **TCP** delivers bytes in order, retransmitting what got lost.
- **TLS** wraps a TCP connection in encryption.
- **HTTP** speaks structured request/response on top of that.

You can mostly forget about TCP and TLS day to day — but knowing they're there explains a lot
of behaviour: why opening a slow site feels "stuck" before any pixels appear (TCP/TLS handshake),
why mixed-content warnings exist (an HTTPS page must not load HTTP subresources), why
keep-alive matters (re-using a connection for several requests avoids redoing TCP+TLS).

## Reading a status code at a glance

| Code | Meaning |
|---|---|
| 200 | OK — body is what you asked for. |
| 301 / 308 | Permanent redirect. |
| 302 / 307 | Temporary redirect. |
| 304 | Not Modified — your cached copy is fine. |
| 400 | Client error — the request was malformed. |
| 401 / 403 | Unauthenticated / Forbidden. |
| 404 | Not Found. |
| 500 / 502 / 503 | Server-side breakage. |

When something's broken, the status family (`2xx`/`3xx`/`4xx`/`5xx`) usually points you at
the right side of the line — your code or theirs.
