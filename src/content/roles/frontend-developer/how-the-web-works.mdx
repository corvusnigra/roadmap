## Что происходит, когда вы нажимаете Enter

Когда вы вводите `https://example.com/articles` в адресной строке и жмёте
Enter, браузер не достигает сервера магией. Он запускает многошаговый
конвейер, который удобно представить как единую ментальную модель:

1. **DNS** — браузер спрашивает: «Какой IP у `example.com`?» Операционная
   система кэширует ответ на некоторое время; иначе запрашивает DNS-резолвер
   (часто провайдерский или `1.1.1.1`/`8.8.8.8`).
2. **TCP** — браузер открывает надёжное соединение к этому IP на порт 443
   (HTTPS) или 80 (HTTP).
3. **TLS** — для HTTPS браузер и сервер выполняют TLS-рукопожатие, согласуют
   ключи шифрования и проверяют сертификат сервера.
4. **HTTP** — браузер отправляет HTTP-запрос вида:
   ```
   GET /articles HTTP/1.1
   Host: example.com
   Accept: text/html
   ```
5. **Ответ** — сервер возвращает строку статуса, заголовки и тело:
   ```
   HTTP/1.1 200 OK
   Content-Type: text/html; charset=utf-8
   Content-Length: 12048

   <!doctype html>...
   ```
6. **Отрисовка** — браузер парсит HTML, запрашивает все связанные ресурсы
   (CSS, JS, картинки, шрифты) и постепенно рисует страницу.

## Почему важна многослойность

У каждого слоя — своя задача:

- **DNS** — телефонная книга: только сопоставляет имена и адреса.
- **TCP** — доставляет байты по порядку, перепосылая потерянные.
- **TLS** — оборачивает TCP-соединение шифрованием.
- **HTTP** — говорит структурированными запросами/ответами поверх всего этого.

В повседневной работе можно почти забыть про TCP и TLS, но знание, что они
есть, объясняет многое: почему «медленный сайт» висит до появления первых
пикселей (рукопожатия TCP/TLS), зачем существуют предупреждения о смешанном
контенте (HTTPS-страница не должна загружать HTTP-ресурсы), почему важен
keep-alive (повторное использование одного соединения для нескольких запросов
избавляет от повторных TCP+TLS).

## Чтение статус-кода по диагонали

| Код | Значение |
|---|---|
| 200 | OK — тело — это то, что вы запросили. |
| 301 / 308 | Постоянный редирект. |
| 302 / 307 | Временный редирект. |
| 304 | Not Modified — закэшированная копия актуальна. |
| 400 | Ошибка клиента — запрос некорректен. |
| 401 / 403 | Не аутентифицирован / Запрещено. |
| 404 | Не найдено. |
| 500 / 502 / 503 | Поломка на стороне сервера. |

Когда что-то ломается, группа статусов (`2xx`/`3xx`/`4xx`/`5xx`) обычно
сразу подсказывает, на чьей стороне проблема — вашей или их.

## Попробуйте

Откройте вкладку Network в devtools браузера, перезагрузите эту страницу и
посмотрите на первый запрос. Найдите статус-код, `Content-Type` и
`Content-Length`. Это и есть HTTP — за кулисами больше ничего нет.
