## Зачем

Эта роль — самопроверка `pnpm content:sync`. Создана без правки `seed.ts`,
`curriculum.ts` или `scaffold-*.ts`. Только: папка → `_role.json` → 2 MDX → sync.

## Минимальный набор для роли

1. `src/content/roles/<slug>/_role.json` — title, summary, status
2. ≥ 1 MDX-файл со status: published (или draft для разработки)
3. `pnpm content:sync` — локально
4. `pnpm content:sync:prod` — на prod (после vercel env pull)

## Где в стеке

**Уровень 0: Основы.** Понять идею перед тем как добавить вторую тему.
